# 程序型记忆九宫格 — 操作约束模型

程序型记忆回答组织「怎么做事」的问题，按约束力层次（宪法、法律、法理）与消费对象（人类友好、AI友好、规则引擎友好）交织为九宫格。

```
                 人类友好          AI友好          规则引擎友好
                  (可读性)        (可解析)          (可执行)
                 ┌──────────────┬──────────────┬──────────────┐
    宪法         │  Bylaw       │ Specification │  Toolkit     │
   （最高约束）    │  工作章程     │  工程标准      │  工具箱       │
                 ├──────────────┼──────────────┼──────────────┤
    法律         │  Handbook    │  Gallery      │  Platform    │
   （中等约束）    │  工作手册     │  工作案例      │  平台         │
                 ├──────────────┼──────────────┼──────────────┤
    法理         │  Tutorial    │  Essay        │  Example     │
   （低约束）      │  工作教程     │  工作札记      │  示例程序     │
                 └──────────────┴──────────────┴──────────────┘
```

## 按约束力层次

### 宪法层 — 最高约束力

规定基本原则与权责划分，不可轻易变更。定义组织的底线和边界。

| 资产 | 消费对象 | 职责 |
|------|---------|------|
| **Bylaw** 工作章程 | 人类友好 | 治理规则的文档，定义宗旨、权力范围和运作原则。所有人必须遵守的基本法。 |
| **Specification** 工程标准 | AI友好 | 规范化工程实践的文档标准，供 AI 系统解析和验证。定义「什么是正确的」。 |
| **Toolkit** 工具箱 | 规则引擎友好 | 封装治理规则的自动化工具集，供 CI/CD 和规则引擎直接调用。 |

### 法律层 — 中等约束力

系统性的程序规范，指导特定领域如何运作。较宪法更具体，但仍有较高约束力。

| 资产 | 消费对象 | 职责 |
|------|---------|------|
| **Handbook** 工作手册 | 人类友好 | 针对具体岗位或业务的标准化操作指导（SOP），确保工作质量和效率。 |
| **Gallery** 工作案例 | AI友好 | 结构化的案例库，供 AI 参考和类比推理。回答「以前类似情况怎么处理的」。 |
| **Platform** 平台 | 规则引擎友好 | 承载组织运营、协同办公和业务流转的数字空间，将流程自动化。 |

### 法理层 — 低约束力

基本原理与方法论，提供指导和启发但不强制执行。

| 资产 | 消费对象 | 职责 |
|------|---------|------|
| **Tutorial** 工作教程 | 人类友好 | 面向人的学习材料，帮助理解和掌握操作方法。 |
| **Essay** 工作札记 | AI友好 | 关于方法论的总结与思考，指导 AI 的上下文构建。 |
| **Example** 示例程序 | 规则引擎友好 | 可执行的代码示例，演示具体实现方式。 |

## 按消费对象

### 人类友好列

面向人类读者，强调可读性与启发性。

```
Bylaw → Handbook → Tutorial
  ↑          ↓
  治理规则    学习方法
```

这套链路从**约束**（章程）到**操作**（手册）再到**理解**（教程），逐层降低约束、增加教育性。

### AI友好列

面向 AI 系统，强调结构化与可解析性。

```
Specification → Gallery → Essay
     ↑             ↓
  结构定义       案例参考    方法论思考
```

AI 从标准中学习规则，从案例中学习模式，从札记中学习判断。

### 规则引擎友好列

面向自动化系统，强调可执行性。

```
Toolkit → Platform → Example
   ↑          ↓
工具集       自动化平台    可运行示例
```

从底层工具到上层平台再到具体示例，构成完整的自动化执行链。

## 三列之间的流转

```
                    发现规律
  人类实践 ─────────────────→ AI 认知
     ↑                          │
     │                   抽象成规则
     │                          ↓
     │                   规则引擎执行
     │                          │
     └──────────────────────────┘
         验证与反馈
```

操作知识从人类实践中来（手册、教程），经 AI 抽象为结构化认知（标准、案例），最终沉淀为可执行规则（工具箱、平台），再通过示例验证后反馈回人类实践。
